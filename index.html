Projects done by me

TheStash

Project done in: Apr 2022 - Apr 2022

I created a forum-like website named "TheStash" to implement 3-tier web architecture with Azure Cloud services.

I received this project during my poly internship days at NCS. Then, my supervisor tasked me and my fellow group of friends, to come up with a website of any functionality and implement it through Azure's powerful cloud platform in a 3-tier web architecture which comprises of following 3 components:

  1. Client frontend website
  2. Server hosting website
  3. Database handling website data

The website has to be complete with full CRUD operations (CREATE,READ,UPDATE and DELETE) for users to operate. So I brainstormed and decided that I want to create a forum based website similar to stackoverflow, and began building it.

I made use of Visual Studio Code as my IDE (Integrated Development Envrionment) for coding, and coded the website in HTML, Javascript, CSS and Bootstrap language all in one Windows VM. Utilized XAMPP and MySQL workbench for development and testing, and used Github to work at home and version control my web application.

After I had finished coding my website, I created a storage account at Azure Cloud, create and pulled my codes from Github to a local repository on Azure and began hosting it on Azure WebApp!

Optimid

Project done in: Sept 2022 - Nov 2022

I worked on a dashboard website to provide users with graph metrics to visualize their electrical appliances utilization rate.

This project was assigned to me when I was in the last semester of my final year in Polytechnic. The idea was to design and work out an IoT environment on AWS Cloud, which my team had chosen smart household appliances to begin with.

Our idea was to visualize and inform our users about their smart home appliances power usage, so that they can make adjustments if they find out any appliances has high power consumption rate.

The design of our simple project is as such, we break our design into 3 parts:

  1. Client side smart home appliance simulated by IFTTT app
  2. Server side AWS EC2 instance serve as:
    • Iot Sensor receiving IFTTT dweet.io messages which sends to MQTT
    • MQTT then acts as the broker which sends out messages by topics of interest
    • Subcriber of the topic then receives the messages and inputs data into the Database for the website

For my part I mainly worked on the design of the website and the input of data from the website and database.
